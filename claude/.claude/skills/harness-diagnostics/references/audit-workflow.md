# Audit 모드 워크플로우

> 프로젝트의 현재 에이전트 친화도를 평가하기 위한 워크플로우.

---

## Phase 1: 프로젝트 탐색

프로젝트의 전반적인 구조와 상태를 파악한다.

### 에이전트 지시 파일 읽기

- `AGENTS.md`, `CLAUDE.md`, `README.md` 내용 분석
- 지시 파일의 완성도와 정확성 평가

### 디렉토리 구조 스캔

- 소스 코드, 테스트, 문서, 설정 파일 구조 파악
- 표준 레이아웃 준수 여부 확인

### 기술 스택 및 프로젝트 유형 식별

- 의존성 파일 기반 언어/프레임워크 파악
- 프로젝트 분류 (web app, library, CLI, monorepo 등)

### Git 히스토리 활동 패턴 확인

- 최근 커밋 빈도 및 패턴 분석
- 주요 기여자 및 활동 영역 파악
- 브랜치 전략 확인

---

## Phase 2: 12원칙 평가

각 원칙을 정량적으로 평가한다.

### 점수 부여

- 12개 원칙 각각에 0-10 점수 부여
- 구체적 근거와 함께 기록
- 점수 기준: `references/principles.md` 참조

### 차원 점수 산출

- 4개 차원별 평균 점수 계산
- 약점 차원 식별
- 계산 방법: `references/maturity-framework.md` 참조

### 성숙도 레벨 결정

- 종합 점수로 L1-L5 등급 결정
- 차원 간 편차 분석

---

## Phase 3: 체크리스트 실행

대상에 맞는 체크리스트를 실행한다.

### 코드베이스 대상

- `references/codebase-checklist.md` 실행 (84항목)
- 카테고리별 통과율 산출

### Skill 대상

- `references/skill-checklist.md` 실행 (40항목)
- 카테고리별 통과율 산출

### 코드 샘플링 방법

체크리스트 외에 코드 품질을 파악하기 위한 샘플링:

| 방법 | 확인 대상 | 수행 방법 |
|------|-----------|-----------|
| **Import 패턴 분석** | 의존성 구조, 모듈 응집도 | 주요 파일 10개의 import 구조 확인 |
| **네이밍 컨벤션** | 변수/함수/파일명 일관성 | 3개 디렉토리의 네이밍 패턴 비교 |
| **파일 크기 분포** | 과도하게 큰 파일/모듈 | 상위 10개 파일 크기 확인 |
| **테스트 커버리지** | 테스트 파일 비율 | 테스트 파일 수 / 소스 파일 수 |
| **Dead code 지표** | 미사용 코드 | 미사용 import, 주석 처리된 코드 탐지 |

## Phase 4: 리포트 생성

평가 결과를 종합하여 리포트를 생성한다.

### Audit 리포트 생성

- `references/report-template.md`의 Audit 리포트 형식 사용
- 12원칙 점수, 차원 점수, 종합 점수 포함
- 체크리스트 실행 결과 포함

### 빠른 개선 항목 섹션

- 적은 노력으로 큰 효과를 얻을 수 있는 항목 top 5
- 구체적 실행 방법과 예상 소요 시간 명시

### 개선 로드맵

- 단기 (1-2주) / 중기 (1-2개월) / 장기 (3개월+) 분류
- 각 과제의 우선순위와 기대 효과 명시
- 다음 audit까지의 목표 점수 설정

### 리포트 형식 검증

리포트 생성 후 출력 전 다음 사항을 검증한다:

- [ ] 공통 헤더(진단 대상, 모드, 날짜, 기술 스택, 진단 범위)가 포함되어 있는가
- [ ] 12원칙 점수가 모두 기재되고, 각각에 근거가 제시되어 있는가
- [ ] 차원 점수 계산이 정확한가 (소속 원칙 점수의 산술 평균)
- [ ] 종합 점수 계산이 정확한가 (`(DimA×0.3 + DimB×0.3 + DimC×0.2 + DimD×0.2) × 10`)
- [ ] 종합 등급(L1-L5)이 점수 범위와 일치하는가
- [ ] 체크리스트 결과 테이블의 합계가 정확한가
- [ ] 자체 점검 부록이 포함되어 있는가
